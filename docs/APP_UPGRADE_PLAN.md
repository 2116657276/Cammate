# LiveAICapture 功能完善文档（场景识别驱动版）

## 1. 目标

在不接入真实 AI 大模型的前提下，把 App 做成可独立完成拍前指导的版本：

- 用户流程：打开相机 -> 开启引导 -> 自动识别场景 -> 获得构图建议 -> 倒计时拍照保存。
- 去掉风格参数，避免过多无效选择。
- 场景识别驱动引导，后续可平滑替换为 YOLO/TFLite/云端视觉模型。

## 2. 版本边界

本次实现（V0.3）：

- 无风格控制（style 完全移除）
- 拍摄模式：`自动` / `人像` / `通用`
- 场景识别：`portrait/general/landscape/food/night`
- 本地引导引擎按场景出建议
- 后端场景检测可插拔：`heuristic`（默认）/`yolo`（可选）

暂不实现：

- 真实第三方大模型推理链路
- 用户体系、云端历史
- 重风格迁移和后处理滤镜

## 3. 信息架构

### 3.1 相机页（主流程）

- 顶部：状态 + 设置入口
- 中部：预览 + Overlay（网格/目标点/bbox）
- 信息卡：
  - 当前建议
  - 识别场景 + 置信度
  - 当前模式
  - 会话计数
- 底部控制：
  - 引导开关
  - 模式切换（自动/人像/通用）
  - 前后摄
  - 闪光模式
  - 倒计时拍照

### 3.2 设置页（配置）

- 引导来源（本地 / 云端API）
- 拍摄模式（自动/人像/通用）
- 分析间隔
- 语音开关
- Debug 开关
- Server URL（预留）
- 恢复默认

## 4. 核心行为定义

### 4.1 场景识别

- 输入：取景帧（JPEG bytes）
- 输出：`scene + confidence`
- 默认：轻量规则模型（亮度/饱和度/中心肤色特征）
- 可选：YOLO（通过环境变量开启）

### 4.2 引导策略

- 人像：中心构图 + 主体框 + 眼睛位置建议
- 通用：三分法优先
- 细分场景（风景/美食/夜景）：在通用基础上细调目标点与文案

## 5. 数据模型与持久化

持久化字段（DataStore）：

- `guide_provider`: `local` | `cloud`
- `capture_mode`: `auto` | `portrait` | `general`
- `server_url`
- `interval_ms`
- `voice_enabled`
- `debug_enabled`

## 6. 验收标准

1. 不依赖后端也能稳定给出场景化建议。
2. 主界面无风格入口，模式切换清晰。
3. 自动识别场景可在 UI 中观察并随取景变化。
4. 倒计时拍照与相册保存闭环可用。
5. 设置持久化，下次启动可恢复。

## 7. 后续 AI API 接入点

### 7.1 Android

- 保持统一事件模型 `AnalyzeEvent`（含 `scene/strategy/target/ui/param/done`）
- 当前本地引擎分支可替换为云端分支，UI 层无需改动

### 7.2 Backend

- 继续沿用 NDJSON 事件顺序
- 可将 `heuristic/yolo` 替换为真实视觉模型或多阶段 pipeline
